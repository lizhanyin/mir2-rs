# 序列帧动画系统

## 概述

本文档描述 Bevy 引擎中序列帧动画（Sprite Sheet Animation）的实现方案，重点解决高帧率显示器下的动画流畅性问题。

## 问题分析

### 序列帧动画的本质

序列帧动画使用 **Timer + 帧切换** 的方式：

```rust
fn animate_sprite(
    time: Res<Time>,
    mut query: Query<(&AnimationIndices, &mut AnimationTimer, &mut Sprite)>,
) {
    for (indices, mut timer, mut sprite) in &mut query {
        timer.tick(time.delta());
        if timer.just_finished() && let Some(atlas) = &mut sprite.texture_atlas {
            atlas.index = if atlas.index == indices.last {
                indices.first
            } else {
                atlas.index + 1
            }
        }
    }
}
```

### 无法真正"插帧"的原因

| 对比项 | 3D 动画 | 2D 序列帧 |
|--------|---------|-----------|
| 数据结构 | 骨骼 + 权重 | 离散图片 |
| 插值方式 | 骨骼位置/旋转插值 | 无中间状态 |
| 帧率适应 | 自动平滑 | 帧数固定 |

**结论**：如果动画是 12fps，游戏运行在 144fps，每个动画帧会显示约 12 个游戏帧。这是序列帧动画的固有限制。

---

## 推荐方案：固定时间步长 + 渲染插值

### 核心思路

分离"游戏逻辑"和"渲染显示"：

```
固定时间步长：
┌─────────────────────────────────────────────────────┐
│  逻辑更新：|────|────|────|────|────|  (固定 60fps) │
│  渲染帧：  |─|─|─|─|─|─|─|─|─|─|─|─|  (任意帧率)   │
│                                                     │
│  ✅ 逻辑稳定：动画速度、物理计算始终一致            │
│  ✅ 网络同步：所有客户端计算结果相同                │
│  ✅ 回放系统：可以精确重现游戏过程                  │
└─────────────────────────────────────────────────────┘
```

### 渲染插值的作用

```
没有插值（卡顿感）：
位置: ●────────────────●────────────────●
      0ms             16ms             33ms
      (位置突然跳跃)

有插值（平滑）：
位置: ●─●─●─●─●─●─●─●─●─●─●─●─●─●─●─●─●
      每个渲染帧都有中间位置
      (视觉上非常流畅)
```

### 效果对比

| 场景 | 传统方式 | 固定步长+插值 |
|------|----------|---------------|
| 60Hz 显示器 | 正常 | 正常 |
| 144Hz 显示器 | 动画太快 | ✅ 速度一致，更流畅 |
| 帧率波动 | 卡顿 | ✅ 视觉平滑 |
| 网络同步 | 不一致 | ✅ 确定性计算 |
| 低端设备 | 慢动作 | ✅ 速度不变（可能跳帧） |

### 2. 位置插值示例

```rust
// 组件定义
#[derive(Component)]
struct Position(Vec3);

#[derive(Component)]
struct PreviousPosition(Vec3);

// 逻辑层（FixedUpdate，稳定运行在 60fps）
fn fixed_movement(
    time: Res<Time>,
    mut query: Query<(&mut Position, &mut PreviousPosition)>,
) {
    for (mut pos, mut prev_pos) in &mut query {
        prev_pos.0 = pos.0;  // 保存旧位置
        pos.0.x += 100.0 * time.delta_seconds();  // 更新新位置
    }
}

// 渲染层（Update，可以是任意帧率）
fn render_interpolate(
    fixed_time: Res<Time<Fixed>>,
    mut query: Query<(&Position, &PreviousPosition, &mut Transform)>,
) {
    let alpha = fixed_time.overstep_fraction();  // 0.0 ~ 1.0

    for (pos, prev_pos, mut transform) in &mut query {
        // 在两个逻辑帧之间平滑插值
        transform.translation = prev_pos.0.lerp(pos.0, alpha);
    }
}
```

## 处理不同帧数的动画

### 问题

传奇2中每个动作的帧数不同：
- 站立：4 帧
- 走路：8 帧
- 攻击：10 帧
- 施法：12 帧

### 解决方案：时间驱动，而非帧数驱动

**核心公式**：
```
progress += delta_time / animation_duration
current_frame = floor(progress * frame_count)
```

### 数据结构设计

```rust
/// 动画定义
#[derive(Clone, Resource)]
struct AnimationConfig {
    /// 动作类型
    action: Action,
    /// 该动作的总帧数
    frame_count: usize,
    /// 该动作的总时长（秒）
    duration: f32,
    /// 帧索引在图集中的起始位置
    start_index: usize,
    /// 是否循环
    looping: bool,
}

impl AnimationConfig {
    /// 根据时间进度计算当前应该显示哪一帧
    fn get_frame_at(&self, progress: f32) -> usize {
        // progress: 0.0 ~ 1.0
        let frame = (progress * self.frame_count as f32).floor() as usize;
        // 防止越界
        frame.min(self.frame_count - 1)
    }

    /// 每帧的持续时间
    fn frame_duration(&self) -> f32 {
        self.duration / self.frame_count as f32
    }
}
```

### 角色动作定义

```rust
/// 角色动作
#[derive(Clone, Copy, PartialEq)]
enum Action {
    Stand,      // 站立 4帧
    Walk,       // 走路 8帧
    Run,        // 跑步 8帧
    Attack,     // 攻击 10帧
    Cast,       // 施法 12帧
    Death,      // 死亡 6帧
}

/// 动画配置表
impl Action {
    fn config(&self) -> AnimationConfig {
        match self {
            Action::Stand => AnimationConfig {
                action: *self,
                frame_count: 4,
                duration: 0.8,    // 800ms 完成整个动画
                looping: true,
                start_index: 0,
            },
            Action::Walk => AnimationConfig {
                action: *self,
                frame_count: 8,
                duration: 0.6,    // 600ms 完成一步
                looping: true,
                start_index: 4,
            },
            Action::Attack => AnimationConfig {
                action: *self,
                frame_count: 10,
                duration: 0.5,    // 500ms 完成攻击
                looping: false,
                start_index: 20,
            },
            // ... 其他动作
        }
    }
}
```

### 动画状态组件

```rust
/// 角色动画状态组件
#[derive(Component)]
struct AnimationState {
    current_action: Action,
    config: AnimationConfig,
    /// 动画播放进度 0.0 ~ 1.0
    progress: f32,
    /// 是否循环
    looping: bool,
}
```

### 系统实现

```rust
/// 固定时间步长更新动画逻辑
fn update_animation_logic(
    time: Res<Time>,
    mut query: Query<&mut AnimationState>,
) {
    for mut state in &mut query {
        // 根据动画时长计算进度增量
        let delta_progress = time.delta_seconds() / state.config.duration;
        state.progress += delta_progress;

        // 处理循环或完成
        if state.progress >= 1.0 {
            if state.looping {
                state.progress = state.progress % 1.0;
            } else {
                state.progress = 1.0;  // 停在最后一帧
            }
        }
    }
}

/// 渲染层：根据进度计算当前帧
fn render_animation(
    mut query: Query<(&AnimationState, &mut Sprite)>,
) {
    for (state, mut sprite) in &mut query {
        if let Some(atlas) = &mut sprite.texture_atlas {
            // 根据进度计算帧索引
            let frame = state.config.get_frame_at(state.progress);
            atlas.index = state.config.start_index + frame;
        }
    }
}

/// 动作切换处理
fn handle_action_change(
    mut query: Query<(&PlayerInput, &mut AnimationState)>,
) {
    for (input, mut state) in &mut query {
        let new_action = if input.attacking {
            Action::Attack
        } else if input.running {
            Action::Run
        } else if input.moving {
            Action::Walk
        } else {
            Action::Stand
        };

        // 动作改变时重置动画
        if state.current_action != new_action {
            state.current_action = new_action;
            state.config = new_action.config();
            state.progress = 0.0;  // 从头开始播放
        }
    }
}
```

## 总结

| 方面 | 处理方式 |
|------|----------|
| 帧数不同 | 用**时间进度**而非帧计数 |
| 动画速度 | 每个动作配置独立的 `duration` |
| 动作切换 | 重置 `progress = 0.0` |
| 帧率无关 | 固定时间步长 + delta time |
| 高刷新率 | 逻辑固定，渲染插值 |

这种方案的优势：
1. **稳定性**：游戏逻辑与帧率解耦，不同硬件体验一致
2. **可扩展性**：配置驱动，易于添加新动画
3. **网络友好**：确定性计算为未来网络同步打基础
4. **流畅性**：渲染插值让高刷新率显示器体验更好

```rust
fn setup_character(
    mut commands: Commands,
    mut library: ResMut<AnimationLibrary>,
    frames: Vec<Handle<Image>>,
) {
    // 添加动画剪辑到库
    library.add_clip(
        AnimationClip::new("stand", AnimationAction::Stand, 4, 0.8, 0)
            .with_frames_per_direction(4)
    );
    library.add_clip(
        AnimationClip::new("walk", AnimationAction::Walk, 8, 0.6, 32)
            .with_frames_per_direction(8)
    );
    library.add_clip(
        AnimationClip::new("attack", AnimationAction::Attack, 10, 0.5, 160)
            .with_frames_per_direction(10)
            .with_looping(false)
    );

    // 创建角色实体
    let state = SpriteAnimationState::new("stand")
        .with_frames(frames)
        .with_direction(AnimationDirection::Down);

    commands.spawn((
        Sprite::default(),
        Transform::default(),
        Visibility::default(),
        state,
    ));
}

fn update_character_animation(
    mut query: Query<&mut SpriteAnimationState, With<Player>>,
    library: Res<AnimationLibrary>,
    input: Res<ButtonInput<KeyCode>>,
) {
    for mut state in query.iter_mut() {
        // 根据输入切换动画
        if input.pressed(KeyCode::KeyW) {
            state.play("walk", &library);
            state.direction = AnimationDirection::Up;
        } else if input.pressed(KeyCode::KeyS) {
            state.play("walk", &library);
            state.direction = AnimationDirection::Down;
        } else {
            state.play("stand", &library);
        }
    }
}
```

#### AnimationClip 字段与 Mir2 TActionInfo 对应关系

```pascal
// 原始 Mir2 的 TActionInfo 结构
TActionInfo = packed record
    start   :Word;  // 开始帧 -> start_index
    frame   :Word;  // 帧数 -> frame_count
    skip    :Word;  // 帧间隔 -> skip
    ftime   :Word;  // 每帧延迟时间（毫秒）-> duration
    usetick :Word;  // 用于行走动画
end;
```

| Mir2 字段 | AnimationClip 字段 | 说明 |
|-----------|-------------------|------|
| `start` | `start_index` | 帧索引在图集中的起始位置 |
| `frame` | `frame_count` | 动画总帧数 |
| `skip` | `skip` | 帧间隔（每帧之间跳过的索引数） |
| `ftime` | `duration` | 每帧延迟时间，duration = frame * ftime / 1000 |

#### 使用 from_action_info 创建动画

```rust
// 从 Mir2 的 TActionInfo 参数直接创建动画剪辑
```
